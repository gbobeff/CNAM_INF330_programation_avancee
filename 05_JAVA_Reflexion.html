Définition

  • En programmation informatique, la réflexion est la capacité d'un programme à examiner, et éventuellement à modifier, ses propres structures internes de haut niveau lors de son exécution.(Wikipedia)
    • l'introspection, qui est la capacité d'un programme à examiner son propre état
    • l'intercession, qui est la capacité d'un programme à modifier son propre état d'exécution ou d'altérer sa propre interprétation ou signification.

Principe

  • en Java on doit créer un objet métier et accéder aux variables d'instance ou appeler la méthode de cet objet via l'opérateur point ( . )
    • dans ce cas on manipule les objets du modèle
  • la réflexion permet d'accéder à un champ d'un objet si on connait le nom d'un tel champ ou appeler une méthode de l'objet si on connait le nom de la méthode, les types de ses paramètres et les valeurs des paramètres à transmettre
    • dans ce cas on manipule les objets à travers du metamodèle
  • Java Reflection (nom de l'API en Java) :
    • permet d'accèder à des membres (variables et méthodes) n'importe où dans l'application
    • limitation : pas de modification des classes, ex. ajout d'attributs ou de méthodes à l'exécution

        • Classes et méthodes permettant
      • Accès à l'information sur les classes
        • attributs
        • méthodes
        • constructeurs
      • Manipulation des objets de ces classes
        • modification d'attributs
        • appel de constructeurs
        • appel de méthodes

    Membres d'une Class

    • A partir d'une class ont peux accéder à l'ensemble de ces membres :
      • les constructeurs d'objets de type T sont représentés par des instances de la classe Constructor<T>
      • les champs sont représentés par des instances de la classe Field
      • les méthodes sont représentées par des instances de la classe Method
    • ... de manière générale, avec XXX égale à Constructor, Method, Field ou Class, on sait que :
      • la méthode getDeclaredXXXs() renvoie tous les XXX (privés inclus) qui sont déclarés par la classe, c'est-à-dire non hérités
      • la méthode getXXXs() retourne tous les XXX publics, y-compris ceux qui sont hérités
      • la méthode getDeclaredXXX(param) renvoie le XXX déclaré, dont le nom et/ou le type des paramètres est donné en argument
      • la méthode getXXX(param) retourne le XXX public, hérité ou non, dont le nom et/ou les types (objets Class) des paramètres sont donnés en argument
